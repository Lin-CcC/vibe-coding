# 账号状态与功能菜单联动（Design Spec）

日期：2026-03-14  
范围：`front/html` 静态原型（本地存储模拟账号/登录态）

## 背景与问题

当前存在以下问题：

1. 已登录时，侧边功能菜单里的头像状态与首页左上角头像不一致；期望两处状态一致，且仅在未登录时显示“登录”字样。
2. 已登录时，功能菜单底部“关闭”按钮未变更为“退出登录”；并且未登录账号不应显示“关闭/退出登录”按钮（抽屉可通过点击遮罩或点击菜单项关闭）。
3. “切换账号”交互未完成：点击后底部弹出 action sheet，展示历史登录过的账号；点击账号一键切换；同时提供“添加账号”入口跳转到登录页。

## 术语

- **已登录**：`getUser().loggedIn === true`（允许由现有逻辑从本地存储推断）。
- **未登录**：`getUser().loggedIn === false`。
- **历史账号列表**：`getAccounts()`（localStorage）。

## 目标（Goals）

### G1. 头像显示规则统一

- 所有页面中标记为 `[data-user-avatar]` 的头像容器：
  - 已登录：显示用户头像（`avatarDataUrl`）或用户名首字符。
  - 未登录：显示文本“登录”。
- 功能菜单头像（`[data-el="menu-avatar"]`）：
  - 已登录：显示与 `[data-user-avatar]` 相同的头像渲染效果（同一套渲染函数）。
  - 未登录：显示“登录”，点击跳转登录页。

### G2. 功能菜单底部按钮规则

- 未登录：不显示底部按钮（既不显示“关闭”，也不显示“退出登录”）。
- 已登录：底部主按钮显示“退出登录”，点击后：
  - 将当前用户写入历史账号列表（便于下次切换）。
  - 设置 `loggedIn=false` 并触发 UI 刷新事件。
  - 关闭抽屉并回到首页（保持现有行为）。
- 关闭抽屉方式（未登录/已登录均可）：
  - 点击遮罩空白处关闭；
  - 点击抽屉内的链接项自动关闭（保持现有行为）。

### G3. 切换账号 action sheet

- 点击“切换账号”后，从底部弹出 action sheet：
  - 展示 `getAccounts()` 返回的账号列表（头像/昵称/邮箱或提示文案）。
  - 点击某个账号：将其信息写入当前 user（`loggedIn=true`），关闭 sheet + 抽屉，并跳回首页。
  - 在列表底部追加“添加账号”：关闭 sheet + 抽屉，跳转 `login.html`。

## 非目标（Non-goals）

- 不接入真实后端登录、token、网络请求。
- 不新增复杂的账号管理（删除账号、编辑账号等）。
- 不重做整体 UI，只做交互与显示规则修复。

## 影响范围（Files）

- `front/html/assets/app.js`
  - 统一头像渲染逻辑：未登录显示“登录”。
  - 菜单头部与底部按钮显示逻辑：根据 `loggedIn` 切换/隐藏。
  - 确认并完善“切换账号” action sheet 的展示与切换行为。
- `front/html/index.html`
  - 功能菜单底部按钮：将“关闭”改为可被脚本控制的“退出登录”主按钮（或保留结构但可隐藏）。

## 验收标准（Acceptance Criteria）

### 未登录

- 首页左上角头像按钮显示“登录”；打开功能菜单后菜单头像也显示“登录”。
- 功能菜单底部不出现“关闭/退出登录”按钮。
- 点击“切换账号”弹出 action sheet，展示历史账号 + “添加账号”。

### 已登录

- 首页左上角显示用户头像/首字母；功能菜单头像一致。
- 功能菜单底部显示“退出登录”，点击后退出并回到首页；未登录后底部按钮消失。
- 点击“切换账号”能一键切换到已登录过的账号并回到首页；“添加账号”跳转到登录页。

## 回归点

- 已有的抽屉打开/关闭逻辑不被破坏（遮罩点击、菜单项点击仍可关闭）。
- 登录页点击登录后仍能保存历史账号并回到首页。
